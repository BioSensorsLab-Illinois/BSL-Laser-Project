# review-loop

Use this skill after any major milestone or before claiming a workstream is complete.

Rules:

- use another agent or a clearly separate review pass when available
- keep the review objective and independent from the implementation pass
- review at least:
  - safety and permission conflicts
  - firmware logic stability
  - GUI interaction logic
  - rendered-page quality
  - validation completeness

If meaningful issues are found:

- restart another implementation/validation cycle
- record the findings in the active `Status.md`
- do not close the work just because the code exists
